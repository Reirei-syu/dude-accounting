## 关键决策
- 账套备份不再依赖已结账期间；备份期间选择器允许显示全部期间，若账套暂无期间则按“未设置期间”创建备份。
- 备份主流程从“整库快照”切换为“账套级备份包”，历史整库快照只保留 legacy 校验/删除能力。
- 本轮同时交付“备份包导入为新账套”，不支持覆盖已有账套。
- 删除账套前的备份与电子档案导出都改为强提示，不再做硬阻断，但必须要求显式风险确认并写入操作日志。
- 审计执行采用真实 Electron + `electron-vite --remoteDebuggingPort` + Python Playwright CDP，不走浏览器预览假环境。
- 为避免原生保存对话框阻塞自动化，报表导出验证沿用同一 IPC 通道但显式传入文件路径。
- 当前开发库已写入多组 `自动测试-*` 账套作为调试与审计痕迹，未自动清理。
- 打印链路修复采用“预览页即恢复入口”策略：在预览工具栏中提供缩放与紧凑模式，不新增新的业务 IPC。
- 报表默认口径修复采用“显式说明 + 成功文案提示”策略，不改变默认业务口径。
- 打印方向选择采用“预览页方向控件 + `print.print/exportPdf` 参数覆盖”方案，避免为 job 状态引入新的持久化字段。
- 报表头部收敛同时作用于打印预览和报表导出 HTML，统一只保留“编制单位、会计期间、单位”。
