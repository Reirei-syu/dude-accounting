## 关键决策
- 打包版用户数据与设置必须独立于安装目录，升级或改安装路径都不得影响已有数据。
- 兼容迁移方向定为“旧安装目录 -> userData”，且仅在目标库不存在时执行。
- 当旧安装目录库与 legacy `userData` 根目录库同时存在时，按主库及其 `-wal/-shm` 辅助文件的最新修改时间选取迁移源，避免拿到较旧副本。
- 默认日志目录也应脱离安装目录，避免升级覆盖与权限耦合。
