# 会计科目现金流量标记优化方案

## 1. 背景
- 当前会计科目设置页已支持维护“现金流量科目”布尔字段，但科目列表未直接展示该状态。
- 当前子科目创建、编辑时允许独立修改现金流量标记，未约束“上级科目为现金流量科目时，下级科目必须为现金流量科目”。

## 2. 目标
- 在科目列表中新增“现金流量”列。
- 对标记为现金流量科目的科目显示“是”，否则显示为空。
- 当上级科目为现金流量科目时，新增下级科目时强制继承为现金流量科目。
- 当任一科目被设置为现金流量科目时，其全部下级科目自动同步为现金流量科目。
- 当下级科目的上级已是现金流量科目时，编辑下级科目时不允许取消该标记。

## 3. 设计方案
- 服务层：
  - 在 `accountSetup.ts` 中新增父科目现金流量约束与级联更新逻辑。
  - 创建子科目时，如果上级科目 `is_cash_flow = 1`，则忽略前端传入值并强制写入 `true`。
  - 编辑科目时，如果上级科目 `is_cash_flow = 1`，则当前科目 `is_cash_flow` 强制保持 `true`。
  - 编辑科目时，如果当前科目最终被设置为 `true`，则批量将其全部下级科目同步为 `true`。
- UI 层：
  - `SubjectSettings.tsx` 科目列表新增“现金流量”列。
  - 新增/编辑弹窗中，当当前上级科目已是现金流量科目时，自动勾选并禁用“现金流量科目”复选框，并显示提示文案。
- 测试：
  - 为服务层补充“父级为现金流量时创建子级强制继承”“父级切换为现金流量时子级级联同步”“父级为现金流量时禁止子级取消”测试。

## 4. 涉及模块
- `src/main/services/accountSetup.ts`
- `src/main/services/accountSetup.test.ts`
- `src/renderer/src/pages/SubjectSettings.tsx`

## 5. 数据结构变更
- 无新增表结构。
- 继续复用 `subjects.is_cash_flow` 字段。

## 6. 接口变更
- 无 IPC 协议字段新增。
- `subject:create` 与 `subject:update` 仍沿用现有 `isCashFlow` 字段，但服务端增加强制约束与级联行为。

## 7. 风险评估
- 级联更新可能影响已有下级科目现金流量标记，需要确保只在“父级最终为现金流量科目”时向下同步为 `true`，不做反向清空。
- UI 禁用复选框后，仍需确保服务端兜底，避免绕过前端调用。
