# 企业会计准则报表模板

## 快速使用

- 默认按现行有效的一般企业报表格式输出四大报表，不混入民非、政府会计、事业单位或小企业口径。
- 先读取 `references/official-basis.md` 确认适用依据；如用户追问“最新口径”或“为什么这样列报”，直接引用该文件中的现行依据和变更观察。
- 需要标准行次顺序、JSON 模板或 Excel 友好结构时，读取 `references/report-templates.json`。
- 需要做格式转换、空模板填值、合计校验或附注提示时，读取 `references/output-rules.md`。
- 如需从企业科目体系映射到报表，请联动使用 `$qiye-cas-2026-accounts`，但不要在本技能内臆造科目到报表项目的一对一映射。

## 工作流

1. 先确认范围。
   仅服务 `enterprise` 一般企业。
   默认使用“已执行新金融准则、新收入准则和新租赁准则”的现行报表格式。
   如果用户明确要求历史口径，再说明这是历史/过渡版本，不作为默认模板。

2. 再确认交付物。
   默认输出四大报表全套。
   如果用户只要某一张表，只输出所需报表，但仍保持该表的完整官方行次。
   支持三种输出形态：`markdown`、`json`、`excel-friendly`。

3. 按标准模板生成。
   从 `references/report-templates.json` 取出 canonical 模板，保留原始顺序、标题、列头、总计行和层级。
   不得把附件草稿里的错误项目原样带入正式输出，例如资产端不得出现“合同负债”。

4. 处理数据填充。
   若用户只要空模板，所有金额字段保持空白或 `null`。
   若用户提供试算平衡、科目余额或已归集数据，只填入能明确映射的项目；无法确定的项目保留空白并说明缺失依据。
   不得凭经验补齐未提供的金额。

5. 最后做合规校验。
   资产负债表校验 `资产总计 = 负债和所有者权益总计`。
   利润表校验营业利润、利润总额、净利润、综合收益总额计算链。
   现金流量表校验三类净额汇总和期末现金衔接。
   所有者权益变动表校验“年初 + 本年增减 = 年末”。

## 强制规则

- 只输出一般企业报表，不输出民非“业务活动表”、政府会计报表或其他行业专用报表。
- 默认采用财政部 2019 年一般企业财务报表格式附件 2 口径；截至 2026-03-09，2025 年底发布的修订征求意见稿尚未见正式生效文件。
- 在“完整模板”模式下，不删减官方项目；仅在用户明确要求“按重要性删减不存在项目”时，才允许依据现行制度做删减，并必须说明已删减。
- 保留比较信息列。
- 金额默认单位为“元”；除非用户明确指定，否则不改成“万元”。
- 对于空模板输出，数值字段留空，不填 `0.00`。
- 对于 JSON 输出，遵循 `references/report-templates.json` 的字段名和顺序。
- 对于 Excel 友好输出，每张报表单独一个 sheet，不合并四表到一个 sheet。

## 输出约定

- `markdown`
  适合直接给用户查看。按标题、编制单位、期间、单位、表格主体、附注提示顺序输出。
- `json`
  适合喂给前端报表模块。使用 `metadata`、`columns`、`sections`/`matrix`、`rows`、`formula`、`note_prompts` 等结构。
- `excel-friendly`
  适合后续导出或写入 xlsx。结构与 JSON 一致，但要额外提供建议 sheet 名和表头区域。

## 常见请求处理

- “给我企业资产负债表模板”
  读取 `report-templates.json` 中 `balance_sheet`，输出该表完整模板。
- “给我四大报表 JSON 模板”
  读取整个 `reports` 对象，按原字段输出。
- “把这份企业报表改成符合企业会计准则”
  先比对官方行次，再补齐缺项、纠正错项、修正列头和比较列。
- “给我 Excel 模板字段”
  使用 `excel-friendly` 约定，按每表一个 sheet 组织。
- “最新口径是什么”
  先读取 `official-basis.md`，说明截至 2026-03-09 的现行有效依据；把 2025 年征求意见稿明确标注为“未正式生效”。

## 资源

- `references/official-basis.md`
  现行有效制度依据、官方链接和未来变化观察。
- `references/report-templates.json`
  四大报表的 canonical 模板，供 JSON、Markdown、Excel 友好输出复用。
- `references/output-rules.md`
  输出格式、期间口径、校验链和附注提示规则。

## 失败边界

- 若用户要求政府会计、事业单位、民非或小企业口径，明确拒绝并说明本技能不适用。
- 若用户要求“最新修订正式文本”，但官方仅有征求意见稿，要明确区分“现行有效规则”和“拟修订方向”。
- 若输入数据不足以安全填值，只返回空模板或半填模板，并列出缺失字段。
