# 隐蔽高风险模式清单

## 1. 循环不收敛

- 症状：停止输入后任务数、日志量、重试次数仍持续上涨。
- 常见根因：失败重试缺少上限、状态机缺少终态、补偿动作再次触发原入口。
- 重点看：任务表、重试计数、定时器、消息消费确认、死信队列。

## 2. 累积型重复写入

- 症状：短测不明显，长跑后出现重复记录、重复附件、重复快照。
- 常见根因：幂等键缺失、唯一约束不完整、导入/恢复流程绕过去重。
- 重点看：唯一索引、自然键、去重表、恢复/补录路径。

## 3. 快照或汇总漂移

- 症状：源数据仍对，但报表、缓存、索引、汇总表逐渐偏离。
- 常见根因：增量更新漏边界、批处理失败未补偿、历史数据重放后未重算。
- 重点看：快照生成逻辑、增量游标、重算入口、源表对账。

## 4. 资源泄漏

- 症状：内存、句柄、临时文件、数据库/WAL、日志体积持续上升且不回落。
- 常见根因：文件句柄未关闭、临时目录不清理、长连接/定时器未释放、失败重试写爆日志。
- 重点看：文件系统、长连接、定时任务、日志轮转、数据库 checkpoint。

## 5. 恢复放大

- 症状：重启、恢复、重新导入后积压更严重，或者同一批数据被二次放大处理。
- 常见根因：恢复缺少幂等保护、中间状态回放策略错误、失败任务重复入队。
- 重点看：启动恢复逻辑、恢复脚本、补偿任务、任务去重键。

## 6. 大数据量下的边界崩溃

- 症状：小样本正常，大样本出现分页错误、排序异常、导出卡死、查询超时。
- 常见根因：O(n²) 算法、全表扫描、分页状态依赖前端临时结构、文件遍历无上限。
- 重点看：分页、导出、汇总、排序、目录扫描、跨期查询。

## 7. 误把测试问题当产品问题

- 症状：脚本偶发失败，但系统状态稳定；或环境抖动导致误判。
- 常见根因：断言窗口太短、样本不确定、测试脚本自身有竞态。
- 重点看：测试输入是否固定、断言是否基于最终态、环境噪音是否隔离。

## 8. 推荐排查顺序

1. 先确认是否真的存在“停止输入后仍不收敛”的信号。
2. 再确认问题发生在产品、观测层还是测试层。
3. 最后才决定是否改业务代码；能靠对账面或探针先证实，就不要盲改主逻辑。
