# Skill: 财务软件民间非营利组织账套与报表初始化向导 (NGO-Init-Agent)

## 1. 角色与目标
**角色定位**: 财务软件系统架构级初始化配置助手 (专注民间非营利组织/民办非企业)
**核心目标**: 
1. 依据财政部最新**《民间非营利组织会计制度》（财会〔2024〕25号，2026年起执行）**，为社会团体、基金会、社会服务机构等生成标准的**会计科目表 (Chart of Accounts)**。
2. 建立科目余额与**民间非营利组织财务报表（资产负债表、业务活动表、现金流量表）**之间的映射计算逻辑。
3. 设定具有民间非营利组织特色的业财约束规则（如限定性与非限定性净资产的区分、受托代理业务核算等）。

---

## 2. 知识库：最新民间非营利组织会计科目表标准 (2026版)

系统在创建“民间非营利组织”类型账套时，应默认生成以下 5 大类一级科目。科目编码遵循 `4-2-2` 结构。**注意：凡涉及新制度新增科目已做特别标注。**

### 2.1 资产类 (编号 1xxx)
*流动资产：*
- `1001` 现金
- `1002` 银行存款
- `1009` 其他货币资金
- `1101` 短期投资
- `1102` 短期投资跌价准备
- `1111` 应收票据
- `1121` 应收账款
- `1122` 其他应收款
- `1131` 坏账准备
- `1141` 预付账款
- `1201` 存货
- `1202` 存货跌价准备
- `1301` 待摊费用

*非流动资产及特殊资产：*
- `1401` 长期股权投资
- `1402` 长期债权投资
- `1403` 其他长期投资 *(2026新制度新增)*
- `1421` 长期投资减值准备
- `1501` 固定资产
- `1502` 累计折旧
- `1505` 在建工程
- `1506` 文物资源 *(由原文物文化资产更名)*
- `1509` 固定资产清理
- `1601` 无形资产
- `1602` 累计摊销 *(2026新制度新增)*
- `1701` 长期待摊费用 *(2026新制度新增)*
- `1801` 受托代理资产 *(NGO专属：核算只起中介作用转赠的资产)*

### 2.2 负债类 (编号 2xxx)
- `2101` 短期借款
- `2201` 应付票据
- `2202` 应付账款
- `2203` 预收账款
- `2204` 应付职工薪酬 *(由原应付工资更名)*
- `2206` 应交税费 *(由原应交税金更名)*
- `2209` 其他应付款
- `2301` 预提费用
- `2501` 长期借款
- `2502` 长期应付款
- `2503` 预计负债
- `2601` 受托代理负债 *(NGO专属：与1801强对应)*

### 2.3 净资产类 (编号 3xxx - 核心差异)
- `3101` 非限定性净资产 *(未受时间或用途限制的净资产)*
- `3102` 限定性净资产 *(受时间或用途限制的净资产)*
- `3201` 以前年度净资产调整 *(2026新制度新增)*

### 2.4 收入类 (编号 4xxx)
*(注：系统初始化时，以下所有收入科目必须强制下设“限定性收入”和“非限定性收入”两个二级明细科目)*
- `4101` 捐赠收入
- `4201` 会费收入
- `4301` 提供服务收入
- `4401` 政府补助收入
- `4501` 商品销售收入
- `4601` 投资收益
- `4701` 总部拨款收入 *(2026新制度新增，适用于境外NGO代表机构)*
- `4901` 其他收入

### 2.5 费用类 (编号 5xxx)
*(注：与收入类似，费用类科目也可根据业务需要区分限定性与非限定性，但通常多为非限定性)*
- `5101` 业务活动成本 *(建议启用【项目】辅助核算)*
- `5201` 税金及附加 *(2026新制度新增)*
- `5301` 管理费用
- `5401` 筹资费用 *(可下设“募捐成本”明细)*
- `5501` 资产减值损失 *(2026新制度新增)*
- `5601` 所得税费用 *(2026新制度新增)*
- `5901` 其他费用

## 3. 财务报表初始化与映射规则 (核心算法逻辑)

民间非营利组织的报表结构与企业有显著不同，系统需预置《会民非01表》和《会民非02表》的专属模板。

### 3.1 资产负债表 (会民非 01 表)
*特殊项目映射规则：*
- **文物资源** = `1506` 借方余额
- **无形资产净值** = `1601` - `1602` (新增累计摊销扣减) - 无形资产减值准备
- **受托代理资产** = `1801` 借方余额 (列在资产总计上方单独列示)
- **受托代理负债** = `2601` 贷方余额 (列在负债合计上方单独列示)
- **净资产合计** = `3101` + `3102`

### 3.2 业务活动表 (会民非 02 表 - 替代企业利润表)
**注意：该表结构为多列式，分为“非限定性”、“限定性”和“合计”三列。**

*收入计算：*
- 分别取 `4101` 到 `4901` 各科目的【发生额】。
- 必须根据明细科目（限定性/非限定性）的发生额，分别填入对应列。

*费用计算：*
- 分别取 `5101` 到 `5901` 各科目的【发生额】，填入对应列。
- **业务活动成本**需取 `5101` 的发生额。

*净资产变动额（底行公式）：*
> 净资产变动额 = 收入合计 - 费用合计 + 限定性净资产转为非限定性净资产 (取3101/3102结转发生额) + 以前年度净资产调整

## 4. 账套初始化与业务逻辑约束规则 (系统防呆设计)

执行民间非营利组织账套初始化及日常记账时，后台逻辑需触发以下校验机制：

1. **受托代理业务强制平衡校验**
   - 任何涉及 `1801 受托代理资产` 的凭证或期初余额，必须有等额的 `2601 受托代理负债`。
   - `资产负债表`中，受托代理资产期末余额必须等于受托代理负债期末余额。
2. **净资产结转逻辑 (期末结转配置)**
   - 月末/年末结账时，系统自动结转功能应预置两套规则：
     - **非限定性结转**：将所有收入/费用科目的“非限定性”明细余额结转至 `3101 非限定性净资产`。
     - **限定性结转**：将所有收入/费用科目的“限定性”明细余额结转至 `3102 限定性净资产`。
3. **“净资产解除限制”专门通道**
   - 当达到限定条件（如项目完工、时间到期）时，系统需提供专用的记账模板：
     - 借：限定性净资产 (`3102`)
     - 贷：非限定性净资产 (`3101`)
4. **辅助核算强制项**
   - 初始化时，系统应提示用户为 `5101 业务活动成本` 和 `4101 捐赠收入` 强制启用【项目核算】（反映具体公益项目收支）和【捐赠人/供应商/客户核算】，以满足审计对专款专用的合规要求。
5. **固定资产与无形资产折旧/摊销扩展**
   - 根据新制度，初始化时需激活【无形资产摊销模块】，支持对 `1601` 自动生成转入 `1602` 的摊销凭证。
   - 特别注意：**文物资源 (`1506`) 不计提折旧**，系统应将其排除在折旧引擎之外。

## 5. 新旧账套无缝衔接扩展 (针对 2025 -> 2026 升级用户)
*(系统一键升级插件逻辑)*
如果用户从2004版旧制度升级至2026版新制度，系统在“结转下年（2026）”时自动执行以下映射：
- 原 `文物文化资产` 余额平移至新科目 `文物资源`。
- 原 `应付工资` 余额平移至新科目 `应付职工薪酬`。
- 原 `应交税金` 余额平移至新科目 `应交税费`。
- 自动为所有净资产历史调整挂载到新增的 `3201 以前年度净资产调整` 科目。
