# Dude Accounting PRD

## 1. 文档目的
本文档用于将当前仓库的真实实现状态、产品边界和下一阶段需求统一下来，避免产品描述与代码实际状态脱节。本文档以当前 Electron + React 单机版项目为准，重点更新“账套设置”模块，尤其是“会计科目设置”和“辅助账设置”。

## 2. 产品定位
- 产品名称：Dude Accounting
- 产品形态：桌面端单机财务软件
- 运行平台：Windows / macOS / Linux
- 目标用户：中小企业财务、记账人员、内部会计管理人员
- 当前定位：以本地账套管理、凭证录入、账务处理和基础账套设置为核心的桌面财务应用雏形

## 3. 当前项目真实状态

### 3.1 已实现
- 登录页与本地账号登录流程
- Electron 主进程 + React 渲染进程 + Preload IPC 桥接架构
- 多账套管理：创建账套、删除账套、切换账套、切换会计期间
- 会计准则模板切换：`enterprise` / `npo`
- 企业账套模板初始化时自动导入完整企业会计准则一级科目
- 凭证录入：科目联想、`=` 自动找平、回车跨列/跨行、现金流量项目必填校验
- 凭证管理：列表、审核、记账、反记账、反审核、删除
- 凭证查询：按日期区间、摘要关键字查询
- 系统参数：允许制单人与审核人为同一人
- 账号管理：管理员新增/删除普通用户，配置权限
- 会计科目设置页面：当前支持按树形展示科目，并可新增基础自定义科目

### 3.2 已有但未完整打通
- `AuxiliarySettings.tsx` 页面已存在
- `src/main/ipc/auxiliary.ts` 已存在
- 但辅助账模块当前未完整接入主进程注册、预加载类型定义和工作区组件映射，用户无法形成稳定可用流程

### 3.3 当前缺口
- 会计科目设置不支持“编辑现有科目”
- 一个科目不能配置多个辅助项
- 系统科目与自定义科目的可编辑边界未在交互层清晰体现
- 辅助账设置缺少完整可用的新增、修改、删除、分类管理闭环
- 文档内容与当前代码状态不一致，且原始文档存在编码不可读问题

## 4. 本轮迭代目标
本轮聚焦“账套设置”模块，目标是把“会计科目设置”和“辅助账设置”补到可实际使用的状态，并同步更新 PRD/TDD。

### 4.1 会计科目设置
- 用户可以查看当前账套全部科目，按类别和父子关系树形展示
- 用户可以选中任意科目查看详情
- 企业账套必须继续遵守企业会计准则一级科目体系
- 系统一级科目名称、编码不可修改
- 自定义明细科目允许修改
- 新建科目时必须选择上级科目，自动继承上级科目的类别、余额方向
- 新建和修改科目时都支持配置辅助项
- 一个科目支持配置多个辅助项
- 支持的辅助项类别至少包括：
  - 客户
  - 供应商
  - 员工
  - 项目
  - 部门
  - 自定义

### 4.2 辅助账设置
- 用户可以按类别查看辅助账档案
- 用户可以新增辅助账
- 用户可以修改辅助账编码和名称
- 若辅助账已被凭证使用，则不允许删除
- 辅助账分类与会计科目中的辅助项配置保持一致

## 5. 核心业务规则

### 5.1 企业会计准则约束
- 对于 `enterprise` 账套，系统必须继续使用完整企业会计准则一级科目模板
- 企业一级科目编码和名称不可被用户修改
- 用户仅可在一级科目下新增或维护自定义明细科目

### 5.2 科目规则
- 科目编码在同一账套内唯一
- 新建明细科目必须选择上级科目
- 明细科目编码应以上级科目编码开头
- 自定义科目默认 `is_system = 0`
- 科目是否启用辅助项由“辅助项类别集合是否为空”决定
- 科目可单独配置是否为现金流量相关科目

### 5.3 辅助账规则
- 辅助账按类别管理，类别内编码唯一
- 删除前必须校验凭证明细是否引用该辅助账
- 辅助账是可被科目引用的档案数据，不直接改变凭证金额逻辑

### 5.4 权限规则
- 科目设置、辅助账设置属于 `ledger_settings` 权限
- 无该权限的用户不能新增、修改、删除账套设置数据

## 6. 用户故事

### 6.1 财务主管
作为财务主管，我希望在新建账套后直接获得企业会计准则标准科目，并在需要时新增企业自己的明细科目，以便保持账套规范统一。

### 6.2 记账人员
作为记账人员，我希望在维护科目时可以勾选多个辅助项，例如“客户 + 项目”，这样后续凭证录入才能支持更细的核算维度。

### 6.3 管理员
作为管理员，我希望能独立维护辅助账档案，例如客户、部门、项目等，以便账套基础资料能长期复用和持续维护。

## 7. 交互要求

### 7.1 会计科目设置页面
- 左侧为科目树
- 右侧为科目详情
- 顶部提供“新增科目”“编辑科目”入口
- 新建/编辑使用独立 Dialog
- Dialog 内可配置：
  - 上级科目
  - 科目编码
  - 科目名称
  - 是否现金流量科目
  - 辅助项多选

### 7.2 辅助账设置页面
- 左侧为辅助账类别列表
- 右侧为该类别下档案列表
- 顶部提供“新增辅助账”
- 新增/编辑使用独立 Dialog
- 列表行提供“编辑”“删除”

## 8. 非功能要求
- 所有账套设置数据保存在本地 SQLite
- 所有写操作通过主进程 IPC 处理
- 文案统一为 UTF-8 中文
- 页面必须兼容当前项目的玻璃态 UI 风格
- 不改变现有凭证录入、凭证管理、登录等已实现模块的行为

## 9. 验收标准

### 9.1 会计科目设置
- 可以打开并展示当前账套科目树
- 可以查看企业一级科目和自定义明细科目
- 可以新增自定义明细科目
- 可以编辑自定义科目
- 系统科目不可改名、不可改编码
- 新建和编辑科目时可增加或减少辅助项
- 保存后页面能正确回显辅助项状态

### 9.2 辅助账设置
- 页面可以正常进入工作区
- 可以新增辅助账
- 可以修改辅助账
- 删除已被凭证使用的辅助账时会被阻止
- 切换类别时列表正确刷新

### 9.3 文档
- `prds/prd.md` 与 `prds/TDD.md` 为 UTF-8 可读文本
- 文档内容与当前项目实现和本轮设计范围一致
