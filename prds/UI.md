

\# Dude Accounting - UI 设计与图层逻辑文档



\## 1. 全局设计规范 (Design System)



\### 1.1 视觉风格

\*   \*\*主体风格：\*\* 毛玻璃效果 (Glassmorphism)，轻量化、通透感。

\*   \*\*核心视效参数：\*\* 

&nbsp;   \*   背景模糊 (`backdrop-filter: blur(10px ~ 20px)`)

&nbsp;   \*   面板背景色：半透明白色至极浅蓝色的渐变 (`rgba(255, 255, 255, 0.4)` 到 `rgba(255, 255, 255, 0.1)`)

&nbsp;   \*   边框：1px 半透明白色实线 (`rgba(255, 255, 255, 0.6)`)，用于增强玻璃边缘的物理质感。

&nbsp;   \*   阴影：柔和的深色或主题色扩散阴影 (`box-shadow: 0 8px 32px 0 rgba(31, 38, 135, 0.1)`)

&nbsp;   \*   圆角 (`border-radius`)：全局采用圆角设计，大容器面板使用 `20px`，按钮/输入框使用 `10px` 到 `15px` 胶囊状圆角。



\### 1.2 字体规范

\*   \*\*字体族：\*\* 推荐使用系统级无衬线字体（如 PingFang SC, Microsoft YaHei, San Francisco）。

\*   \*\*颜色：\*\* 

&nbsp;   \*   主标题/强调文字：纯白 `#FFFFFF`。

&nbsp;   \*   正文/输入框提示文案：半透明白色 `rgba(255, 255, 255, 0.8)` 或 浅灰色，确保在复杂背景下的可读性。

\## 2. 全局图层逻辑 (Z-Axis Layer Architecture)



前端实现必须严格遵守以下图层堆叠顺序（从底层到顶层），确保“悬浮/模糊”交互逻辑的正确呈现：



\### Layer 0: 底层背景层 (Z-Index: -1)

\*   \*\*功能定位：\*\* 承载应用的全局壁纸。

\*   \*\*核心特性（必做）：支持用户自定义替换。\*\*

\*   \*\*默认态：\*\* UI初稿中的卡通手绘风格渐变插画（青/蓝色渐变底，包含金币、小猪存钱罐等元素）。

\*   \*\*图片处理逻辑：\*\* 采用 `background-size: cover` 且 `background-position: center`，确保适配不同分辨率。建议增加一层极弱的全局遮罩（如 `rgba(255,255,255, 0.1)`），防止用户上传过亮/过暗的图片导致上层文字不可读。



\### Layer 1: 主结构面板层 (Z-Index: 10)

\*   \*\*功能定位：\*\* 承载软件骨架。

\*   \*\*包含组件：\*\* 

&nbsp;   \*   左侧导航栏主容器（毛玻璃面板）。

&nbsp;   \*   右侧顶部信息栏、Tab标签栏。

&nbsp;   \*   作业区主容器（毛玻璃面板）。



\### Layer 2: 基础内容与交互层 (Z-Index: 20)

\*   \*\*功能定位：\*\* 承载具体业务功能。

\*   \*\*包含组件：\*\*

&nbsp;   \*   导航栏中的主功能按钮（常态）。

&nbsp;   \*   作业区内的子功能模块按钮（空状态时）。

&nbsp;   \*   具体表单、表格内容（如记账凭证表格）、输入框、文字。



\### Layer 3: 悬浮/交互遮罩层 (Z-Index: 50) - \*基于PRD特殊交互\*

\*   \*\*触发条件：\*\* 在已有Tab打开的状态下，点击左侧主功能按钮时触发。

\*   \*\*包含组件：\*\*

&nbsp;   \*   \*\*局部高斯模糊遮罩：\*\* 仅覆盖当前“作业区 (Workspace)”，不覆盖左侧导航和顶部栏。遮罩颜色略深，模糊度加重。

&nbsp;   \*   \*\*悬浮子功能按钮组：\*\* 在模糊遮罩之上居中显示的毛玻璃按钮组。



\### Layer 4: 全局弹窗与上下文层 (Z-Index: 100+)

\*   \*\*功能定位：\*\* 阻断式交互或极高优先级的提示。

\*   \*\*包含组件：\*\*

&nbsp;   \*   \*\*Context Menu (右键菜单)：\*\* 账簿明细下钻、溯源时的悬浮菜单。

&nbsp;   \*   \*\*Dialog (独立对话框)：\*\* 科目新增修改、辅助账设置等弹窗（需带有全局半透明黑色遮罩，屏蔽底部点击）。

&nbsp;   \*   \*\*系统通知 (Notification)：\*\* 如“网盘授权过期”提示弹窗或右上角Toast。

\## 3. 核心页面图层解构 (Page Breakdown)



\### 3.1 登录界面

\*   \*\*Layer 0:\*\* 全局自定义背景。

\*   \*\*Layer 1:\*\* 居中的毛玻璃大面板。

\*   \*\*Layer 2:\*\* 

&nbsp;   \*   `Dude Accounting` Logo / 标题（白色带发光阴影）。

&nbsp;   \*   账号输入框（毛玻璃内凹效果，白边框）。

&nbsp;   \*   密码输入框。

&nbsp;   \*   登录按钮（带蓝色/紫色渐变高亮与弥散阴影，突出点击引导）。



\### 3.2 主功能界面

\*   \*\*Layer 0:\*\* 全局自定义背景。

\*   \*\*Layer 1:\*\* 

&nbsp;   \*   左侧导航毛玻璃面板（独立）。

&nbsp;   \*   右侧作业区毛玻璃大面板（独立）。

\*   \*\*Layer 2:\*\*

&nbsp;   \*   \*\*左侧：\*\* 主功能按钮列表（选中状态有内发光或边框高亮）。

&nbsp;   \*   \*\*右上：\*\* “账套名称”、“会计期间”纯文字展示（点击可触发Layer 4的下拉列表）。

&nbsp;   \*   \*\*Tab区：\*\* 胶囊状工作标签按钮，激活状态高亮，附带小号 `x` 图标。

&nbsp;   \*   \*\*作业区：\*\* 功能标签1-4等按钮阵列（网格布局居中）。



\### 3.3 记账凭证页面 (挂载于主界面作业区的某个 Tab 下)

\*   \*\*Layer 0 \& 1:\*\* 继承主界面的背景与基础面板。

\*   \*\*Layer 2 (核心表单区):\*\*

&nbsp;   \*   \*\*顶部操作区：\*\* 新建、保存等胶囊形操作按钮，横向排列。

&nbsp;   \*   \*\*表头区：\*\* “记账凭证”大标题；“单位”、“日期”、“字号”散列分布。

&nbsp;   \*   \*\*凭证表格主体：\*\* 

&nbsp;       \*   外边框使用全局的半透明白线，大圆角包裹。

&nbsp;       \*   内部网格线：比外边框更细、透明度更高的白线（如 `rgba(255, 255, 255, 0.3)`），确保不喧宾夺主。

&nbsp;       \*   列划分：摘要、会计科目、借方、贷方。

&nbsp;   \*   \*\*表尾区：\*\* 记账、审核、制单人信息（纯文字）。

\*   \*\*Layer 3/4 (侧边悬浮区):\*\*

&nbsp;   \*   \*\*现金流编辑按钮：\*\* 固定在表格对应行最右侧的外部（带有 `¥` 符号的圆形/小圆角毛玻璃按钮），高亮显示以引起强制校验的注意。

\## 4. UI 动效与状态定义 (States)



前端在实现毛玻璃组件时，需严格区分以下状态：



\*   \*\*默认态 (Normal)：\*\* 标准毛玻璃，1px白色边框。

\*   \*\*悬停态 (Hover)：\*\* 背景透明度稍微降低（变得更白或更亮），阴影扩散半径增加，提升交互反馈。

\*   \*\*激活/选中态 (Active/Selected)：\*\* 增加内部发光效果，或边框颜色从纯白变为主题强调色（如浅蓝色）。

\*   \*\*禁用态 (Disabled)：\*\* 毛玻璃透明度大幅提升，内容颜色变灰暗，移除阴影（如：结账前置条件不满足时的“结账”按钮，或必填项未填时的“保存”按钮）。
