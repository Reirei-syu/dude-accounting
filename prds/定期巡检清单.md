# Dude Accounting 定期巡检清单

## 1. 推荐固定路径

- 开发仓库：`D:\coding\dude accounting\dude-app`
- 安装目录：`D:\Apps\Dude Accounting`
- 安装包输出目录：`D:\coding\completed\dude-app`
- 备份目录：`D:\DudeAccountingData\Backups`
- 导出目录：`D:\DudeAccountingData\Exports`

说明：

- 开发仓库只用于改代码、测试、打包，不直接作为长期自用运行目录。
- 安装目录建议使用独立可写目录，不建议长期装在开发仓库里。
- 打包后默认自动日志目录是安装目录下的 `logs`；如果后续在系统参数页切换过日志保存路径，以软件内显示路径为准。

## 2. 每周巡检

### 2.1 构建与仓库健康

- 检查当前仓库 `git status` 是否存在未预期改动。
- 执行 `npm run typecheck`。
- 执行 `npm test`。
- 执行 `npm run build`。
- 如准备发布，确认 `electron-builder.yml`、`package.json` 版本号和发布脚本未被误改。

### 2.2 安装目录与日志

- 确认安装目录存在且主程序可启动。
- 检查安装目录下 `logs` 或当前自定义日志目录是否存在。
- 读取最近 7 天错误日志，统计：
  - 是否出现未捕获异常
  - 是否出现重复错误
  - 是否出现启动即报错
- 读取最近 7 天运行日志，关注：
  - IPC 连续失败
  - 批量导出/备份/归档异常增多
  - 单次调用耗时明显异常
- 确认超过 1 个月的旧日志是否已被自动清理。

### 2.3 数据与备份

- 确认最近一份账套备份存在。
- 抽查最近一份备份的时间、大小和目录是否正常。
- 如本周有高风险修改，执行一次备份恢复演练。

### 2.4 功能冒烟

- 登录正常。
- 打开默认账套正常。
- 凭证录入页能打开。
- 凭证管理页能打开。
- 报表查询页能打开。
- 备份页能打开。
- 错误日志卡片能显示路径和今日日志状态。

## 3. 每月巡检

### 3.1 数据安全

- 至少做一次完整恢复演练。
- 核对备份目录是否有最近 1 个月的可用备份。
- 检查是否存在长期未校验的备份包。

### 3.2 发布环境

- 重新打一次安装包，确认仍可输出到 `D:\coding\completed\dude-app`。
- 用新版安装包对现有安装执行覆盖升级测试，确认无需先卸载。
- 确认覆盖升级后原有账套数据仍可读取。

### 3.3 空间与清理

- 检查安装目录大小是否异常增长。
- 检查日志目录、备份目录、导出目录大小是否异常增长。
- 检查自动日志留存是否按 1 个月策略生效。

## 4. 发布前专项巡检

- 更新版本号。
- 运行 `npm run typecheck`。
- 运行 `npm test`。
- 运行 `npm run build`。
- 运行 Windows 安装包构建脚本。
- 安装到独立目录进行覆盖升级测试。
- 验证登录、打开账套、凭证、报表、备份、日志设置。
- 验证新版启动后数据库和用户数据不丢失。

## 5. 巡检输出模板

每次巡检输出建议固定为以下 5 段：

1. 本次巡检范围
2. 构建结果
3. 安装与日志结果
4. 备份与数据安全结果
5. 风险、建议和是否需要发布修复版
