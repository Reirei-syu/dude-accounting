# 合规整改计划

> 目标：依据《电子会计档案管理规范》《电子凭证会计数据标准应用指南》《会计软件基本功能和服务规范》《会计信息化工作规范》，建立 Dude Accounting 的合规整改台账，按阶段分模块推进，避免一次性大改带来高回归风险。

## 1. 产品边界

- 软件定位：代理记账企业为委托单位记账用的单机版财务软件
- 支持账套：企业、民间非营利组织
- 不支持：政府会计、事业单位会计
- 当前优先级：Phase 0 至 Phase 4（进行中）

## 2. 整改台账

| 规范来源 | 条款/主题 | 当前现状 | 是否达标 | 整改阶段 | 回归风险 | 验收标准 |
| --- | --- | --- | --- | --- | --- | --- |
| 会计软件基本功能和服务规范 | 已记账凭证不可逆、不得删除和修改关键字段 | 当前保留受控 `unbookkeep` 路径，需持续约束为授权式反记账并完整留痕 | 否 | Phase 1 | 高 | 普通用户无法反记账；仅具备反记账权限的账号可执行，且必须留痕 |
| 会计软件基本功能和服务规范 | 自动审核规则可查询、可校验、可追溯 | 当前无统一规则留痕与日志查询 | 否 | Phase 1/2 | 中 | 关键自动/人工控制规则与执行记录可查询 |
| 会计软件基本功能和服务规范 | 防止电子会计凭证重复入账 | 当前无电子凭证来源记录与去重机制 | 否 | Phase 2 | 高 | 同源电子凭证重复导入会被阻止或提示 |
| 会计软件基本功能和服务规范 | 适配电子凭证会计数据标准并生成入账信息结构化数据文件 | 当前无电子凭证标准解析与结构化入账输出 | 否 | Phase 2/3 | 高 | 可生成结构化入账数据并关联凭证 |
| 会计软件基本功能和服务规范 | 会计资料归档功能、导出电子会计档案接口 | 已支持档案导出、manifest 留存与包级校验 | 是 | Phase 3 | 中 | 可导出档案包、元数据和导出记录 |
| 会计软件基本功能和服务规范 | 用户操作日志查询与导出 | 当前无统一操作日志 | 否 | Phase 1 | 中 | 能按人员、时间、内容查询并导出日志 |
| 会计软件基本功能和服务规范 | 数据容灾和备份功能 | 备份主流程正从系统级数据库快照切换为账套级备份包，目标是支持按账套生成、校验并导入为新账套；历史整库快照仅保留 legacy 校验/删除 | 进行中 | Phase 3 | 中 | 新备份包按账套生成并可导入为新账套，legacy 整库快照不再作为主流程 |
| 会计信息化工作规范 | 会计信息系统嵌入内部控制、自动防范和监控预警 | 当前控制规则分散，无统一留痕 | 否 | Phase 1 | 中 | 核心控制动作有统一日志与失败提示 |
| 会计信息化工作规范 | 电子会计资料形成、收集、整理、归档管理 | 当前无电子会计资料生命周期模型 | 否 | Phase 2/3 | 中 | 原件、结构化结果、归档导出形成闭环 |
| 会计信息化工作规范 | 电子会计资料备份要求 | 备份包已明确升级为账套级传输包，需补齐 manifest、载荷哈希、附件清单与导入校验链路 | 进行中 | Phase 3 | 中 | 备份包带校验信息、创建记录、附件清单与导入校验 |
| 电子凭证会计数据标准应用指南 | 验签/验真、解析、入账、归档全流程 | 当前完全缺失 | 否 | Phase 2 | 高 | 首批票据类型可完成接收、校验、解析、入账留痕 |
| 电子凭证会计数据标准应用指南 | 首批高频票据优先 | 当前未支持 | 否 | Phase 2 | 中 | 数电发票、银行电子回单/对账单可导入记录 |
| 电子会计档案管理规范 | 归档电子会计资料需独立文件与元数据 | 当前无档案元数据模型 | 否 | Phase 3 | 中 | 档案导出包包含元数据和清单 |
| 电子会计档案管理规范 | 真实性、完整性、可用性、安全性要求 | 已支持 manifest 哈希校验、必要文件检查与原始凭证目录数量核对；可读性检查记录待补 | 否 | Phase 3 | 中 | 导出记录、校验摘要和可读性检查结果可追溯 |
| 电子会计档案管理规范 | 在线/离线存储、异地备份、迁移评估 | 当前无系统支持 | 否 | Phase 3 | 低 | 系统能提供备份/导出产物与检查记录，管理动作有据可查 |
| 会计软件基本功能和服务规范 | 会计凭证、账簿、报表显示、打印、版式导出 | 已落地基础财务报表快照生成、查询、删除与 Excel/PDF 另存为导出，支持月份/跨月区间与未记账口径；民非三张报表已切换到会民非 01/02/03 表制式，账簿查询/打印/标准数据导出待续 | 否 | Phase 4 | 高 | 补齐最低账簿报表输出能力 |

## 3. 分阶段实施

### Phase 0：文档与范围校准
- 重写 `prds/prd.md`
- 重写 `prds/PROJECT_SPEC.md`
- 建立本清单文件

### Phase 1：核心账务控制整改
- 操作日志
- 普通用户取消反记账
- 授权式反记账（原因/审批标记/留痕）
- 账套删除强提示与显式风险确认

### Phase 2：电子凭证标准化底座
- 电子凭证文件记录
- 校验/解析状态
- 去重与入账关联
- 首批票据类型接入

### Phase 3：电子档案与备份导出
- 备份包（重构为账套级备份包，legacy 整库快照仅保留校验/删除）
- 恢复校验（已支持恢复前完整性校验）
- 档案导出包（已支持导出后包级校验）
- 元数据与导出记录

### Phase 4：账簿报表与标准输出
- 账簿查询
- 财务报表
- 基础财务报表快照生成、报表查询、删除与 Excel/PDF 另存为导出（已完成）
- 打印与版式导出
- 标准数据导出

## 4. 实施原则

- 先文档、再数据模型、再 IPC、再 preload、再 UI
- 先补失败测试，再写生产代码
- 每个阶段完成后必须跑 `npm run typecheck` 和 `npm test`
- 每个阶段完成后必须更新 `prds/开发日志.md`
